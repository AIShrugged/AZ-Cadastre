# AZ-Cadastre

Помощник инспектора для приёма и проверки пакетов документов первичной государственной регистрации недвижимости в Азербайджане. Система распознаёт документы, извлекает поля и помечает флагами проблемы полноты и согласованности — но **не** выносит автоматический отказ или одобрение; решение всегда за инспектором.

## Language

### Дело и документы

**Дело** (Case):
Один пакет документов, поданный по одному заявлению на первичную регистрацию. Объединяет несколько услуг (регистрация прав, техпаспорт, адрес, оценка). Единица обработки в системе.
_Avoid_: Заявка, package (в UI-терминах), application.

**Документ** (Document):
Логический документ одного типа внутри Дела (например, техпаспорт). Один загруженный PDF может содержать несколько Документов; система режет PDF на Страницы и склеивает соседние страницы одного типа в Документ.
_Avoid_: Файл, скан.

**Страница** (Page):
Одна страница исходного файла — единица классификации. LLM определяет Тип документа по каждой Странице.

**Тип документа** (DocumentType):
Категория документа из фиксированного азербайджанского перечня: `ərizə` (заявление), `şəxsiyyət vəsiqəsi` (удостоверение личности), правоустанавливающий (`sərəncam` / `əmrdən çıxarış` / `arxiv arayışı`), `plan-sxem`, `layihə / memarlıq həlli` (проект), `lisenziya`, `məlumatlandırma ərizəsi` (уведомление), `qəbz` (квитанция), `qiymətləndirmə müqaviləsi` (договор оценки), `texniki pasport`, `müayinə aktı` (акт осмотра).

**Извлечённое поле** (ExtractedField):
Структурное значение, добытое LLM из Документа: ФИО, адрес, № участка, площадь и т.д. Хранит значение, тип, ссылку на Документ/Страницу и уверенность модели.
_Avoid_: Атрибут, реквизит.

### Проверки и результат

**Проверка** (Check):
Правило, применяемое к Делу. В прототипе два класса: **Полнота** (A) и **Согласованность** (C). (B — корректность заполнения, D — Dövriyyə/Ekspertiza — вне скоупа прототипа.)
_Avoid_: Валидация, правило (в UI).

**Полнота** (Completeness, «A»):
Проверка наличия всех обязательных Типов документов для услуги «первичная регистрация».

**Согласованность** (Consistency, «C»):
Ядро системы. Сверка ключевых полей (ФИО, адрес, № участка, площадь) между всеми Документами Дела. Ядро ценности продукта.

**Флаг** (Flag):
Помеченная проблема — отсутствующий документ, расхождение полей или ошибка. Несёт серьёзность и привязку к Документу/полю. Инспектор помечает Флаг как «принято» или «отклонено».
_Avoid_: Ошибка, issue, замечание (используем «Флаг» единообразно).

**Расхождение** (Discrepancy):
Частный случай Флага: одно и то же поле имеет разные значения в разных Документах. Малое отклонение обмера (площадь ≤2%) — норма и Флагом не считается; >2% или несовпадение текста — Флаг.

**Статус Дела** (CaseStatus):
Итог по Делу: `OK` / `Замечания` / `Некомплект`. Выставляется системой, может уточняться инспектором.

**Отчёт** (Report):
Человекочитаемая сводка Дела: статус, список Флагов с привязкой к странице/полю, уверенность модели. В прототипе — веб-представление плюс выгрузка машинного JSON.

**Уверенность** (Confidence):
Оценка модели [0..1] по извлечённому полю или классификации, показывается инспектору рядом со значением.
